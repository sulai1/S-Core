# Copilot Instructions for S-Core Workspace

## Project Context

You are working on **S-Core**, a monorepo containing:
- Frontend: Vue 3 + Quasar SPA (`projects/talktogether/apps/client`)
- Backend: Express.js + TypeORM (`projects/talktogether/apps/server`)
- Shared Libraries: TypeScript packages in `packages/`

The project uses **ESM modules** (ES2022+), **TypeScript** with strict mode, and a **schema-driven API pattern** with OpenAPI.

## Key Architectural Patterns

### 1. OpenAPI-First API Design
- All routes are defined via OpenAPI schemas in `@s-core/talktogether`
- Server handlers are type-safe and auto-generated from schema
- Client receives fully typed method access: `routes['/path'].method()`

Example:
```typescript
// Server: Handler is type-checked against schema
"/auth/login": {
  post: async (req, options) => ({
    success: true,
    user: { id, email, firstName, lastName }
  })
}

// Client: Fully typed call
const result = await routes['/auth/login'].post({ email, password });
```

### 2. Session-Based Authentication
- Express sessions with secure httpOnly cookies
- Guards check `/auth/session` endpoint before route access
- Session data passed via `options.session` parameter in handlers

### 3. Dependency Injection Pattern
- `src/boot/di.ts` (client) exports singleton services: `routes`, `datasource`, `api`
- Server uses `provider.getModule('db')` for database access
- Follows modular, testable architecture

## Common Development Tasks

### When Adding a New Endpoint
1. **Define in schema** (`packages/talktogether/src/schema.ts`)
2. **Implement handler** in `src/serverFactory.ts` with correct request/response types
3. **Create frontend page** in `src/pages/`
4. **Add route** to `src/router/routes.ts`
5. **Call via client**: `await routes['/path'].method(data)`

### When Modifying Authentication
- Update both `/auth/login`, `/auth/logout`, `/auth/session` endpoints together
- Ensure `options.session` is saved/destroyed correctly
- Test guard logic skips unnecessary checks for public routes

### When Debugging Client 404s
1. Verify server is running (port 3000)
2. Check browser Network tab for actual endpoint calls
3. Confirm `src/boot/di.ts` baseUrl matches server location
4. Clear cache (Ctrl+Shift+Delete)

## Code Style & Standards

### TypeScript Rules
- Use **strict mode** (`"strict": true`)
- Prefer **explicit imports** over wildcard imports
- Use **type annotations** for function parameters/returns
- Use **satisfies** for object type validation

```typescript
// ✅ Good
const endpoints = {
  "/auth/login": { post: handler }
} satisfies OpenApiModule<paths, express.Request>;

// ❌ Avoid
const endpoints = { /* implicit typing */ };
```

### Module Imports
- Always use **relative paths** for local imports (e.g., `'src/types'`)
- Use **absolute paths** for workspace packages (e.g., `'@s-core/core'`)
- ESM modules: always include file extensions (`.ts`, `.vue`)

```typescript
// ✅ Good
import { routes } from 'boot/di';
import { Application } from '@s-core/core';

// ❌ Avoid (CommonJS patterns)
const routes = require('./di');
require('@s-core/core');
```

### Vue Component Patterns
- Use **Composition API** with `<script setup lang="ts">`
- Prefer **`q-*` Quasar components** over native HTML
- Use **`v-model`** for two-way binding in forms

```vue
<script setup lang="ts">
import { ref } from 'vue';
const email = ref('');
const handleSubmit = async () => { /* ... */ };
</script>
```

## Performance & Best Practices

### Client-Side
- Use **lazy loading** for page components: `() => import('pages/...')`
- Use route **guards for auth** (not on mount)
- Cache API responses appropriately
- Minimize bundle by importing only needed Quasar components

### Server-Side
- Use **connection pooling** for database
- Validate all inputs before database queries
- Use transactions for multi-step operations
- Log errors with level/context for debugging

### Testing
- Write unit tests for complex business logic
- Mock external dependencies (database, HTTP calls)
- Run `npm test` before commits
- Use **Vitest** for all testing

## Debugging Tips

### Client debugging
- Use **Vue DevTools** browser extension
- Check **Network tab** for API calls and responses
- Use **console.error** logging (LoginPage guard prints errors)
- Clear `.quasar` cache if hot reload breaks

### Server debugging
- Use `npm run debug` with Node debugger
- Check **request logs** in console (logged in dev mode)
- Verify **session middleware** is executed before routes
- Test endpoints with **curl** or **Postman**

### Module/Import errors
- Reinstall deps: `npm run deps:clean && npm run deps:install`
- Check package is in workspaces array in root `package.json`
- Verify file extensions are correct (`.ts`, `.vue`)

## When to Contact Code Owners

- Database schema changes → Coordinate migrations
- API breaking changes → Test all consuming apps
- Package version bumps → Update all dependents
- Deployment → Follow ansible/docker configs
